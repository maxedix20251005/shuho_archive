1 システム概要

目的と範囲 必須

東京都品川区の書道塾サイトとして、塾紹介・師範紹介・コース案内・作品閲覧・問い合わせ導線を提供する。

公開サイトは「印象形成 → 信頼形成 → 比較検討 → 問い合わせ」の順で情報設計する。

主要コンバージョンは問い合わせフォーム送信とし、現スコープでは体験申込ページは作成しない。

管理画面スコープは「作品管理 / 問い合わせ管理 / お知らせ管理」の3画面とする（体験申込管理は対象外）。

システム構成 必須

・公開Webサイト（トップ / 塾について / 師範 / コース / 作品 / FAQ / アクセス / お問い合わせ）

・管理画面（作品管理 / 問い合わせ管理 / お知らせ管理）

・問い合わせフォーム送信機能（メール通知連携を想定）

・HTTPS前提、レスポンシブ対応（PC/SP）

4 機能詳細（主要機能）

問い合わせフォーム（F-008）

入力項目定義 必須
項目名 型 必須 バリデーション 備考
お名前String◯1〜50文字全角・半角可
メールアドレスString◯メール形式、255文字以内確認用入力推奨
電話番号String△数字・ハイフン、10〜13桁任意入力
お問い合わせ種別Enum◯選択必須例: コース / 料金 / 通塾 / その他
お問い合わせ内容Text◯1〜1000文字HTMLタグ無効化
同意チェックBoolean◯未チェック時は送信不可プライバシーポリシー同意
送信後挙動 必須

・送信成功時: 完了メッセージを表示

・送信失敗時: エラーメッセージを表示し再入力導線を維持

・返信目安（1〜2営業日以内）を画面上に明記

作品管理（F-009）

管理項目定義 必須
項目名 型 必須 バリデーション 備考
作品タイトルString◯1〜80文字公開側見出しに使用
カテゴリEnum◯選択必須師範1 / 師範2 / 師範3 / 一般 / こども
レベルString△0〜40文字任意
説明文Text△0〜1000文字改行保持、タグ無効化
画像File◯JPEG/PNG/WebP、最大5MBaltテキスト必須
公開状態Enum◯下書き / 公開未公開は公開側非表示

問い合わせ管理（F-010）

管理項目定義 必須

・一覧項目: 受付日時 / 氏名 / メールアドレス / 種別 / 状態 / 担当

・詳細項目: 本文、管理メモ、状態（new / processing / closed / spam）

・更新操作: 状態更新、管理メモ更新、更新者・更新日時の監査記録

お知らせ管理（F-011）

管理項目定義 必須

・入力項目: タイトル、本文、掲載日、公開状態（draft / published）

・操作: 新規登録、編集、公開/下書き切替、固定表示フラグ設定

・公開品質: 掲載日フォーマット（YYYY.MM.DD）を統一

5 非機能要件

セキュリティ 必須

・全ページHTTPS必須

・フォーム入力値はサーバ側でバリデーションし、XSS/SQLインジェクション対策を実施

・管理画面は認証必須、権限外アクセスを禁止

・CSRF対策、ログイン試行制限を実装

性能・可用性 必須

・主要ページのLCPは2.5秒以内（4G相当）を目標

・CLSは0.1以下を目標

・月間稼働率99.5%以上（計画停止を除く）

・画像最適化（圧縮 / 適切サイズ配信）を実施

対応環境・アクセシビリティ 必須

・対応ブラウザ: Chrome / Safari / Edge / Firefox の最新版とその1つ前のメジャーバージョン

・レスポンシブ対応: 320px〜1440pxの主要幅でレイアウト崩れがないこと

・本文コントラスト比4.5:1以上、タップ領域44px以上

・意味のある画像には日本語altを設定
