1 システム概要

システムの目的 必須
このシステムが解決する課題と提供する価値

東京都品川区の書道塾における「認知」「比較検討」「体験申込」までを一気通貫で支援する。

主目的は、塾の信頼形成（歩み・師範・作品の可視化）と生徒募集（体験申込/問い合わせ獲得）の最大化。

電話依存を減らし、フォーム経由で必要情報を取得することで、事務対応を効率化する。

システム構成図 必須
フロントエンド ↔ バックエンド ↔ データベースの関係を図示

利用者ブラウザ（PC/スマホ）

↓ HTTPS

Webフロント（トップ/塾紹介/師範一覧/作品/募集要項/FAQ/問い合わせ）

↓ API またはフォーム送信

アプリ層（問い合わせ受付、バリデーション、通知連携）

↓

データ保存（問い合わせ台帳、体験申込台帳、流入パラメータ）

＋ 計測基盤（GA4：CTAクリック/フォーム到達/送信完了）

4 機能詳細仕様

機能仕様テンプレート（コピーして使用）

機能名：体験申込フォーム送信 必須
機能ID：F-007

利用者が体験授業の希望内容を入力し、送信できる機能。

送信前に必須/形式バリデーションを実施し、エラー時は該当項目上にメッセージを表示する。

送信成功時は完了ページへ遷移し、同時に管理者通知（メール等）を実行する。

二重送信防止として、送信中はボタンを無効化する。

入力項目一覧 必須
項目名データ型必須バリデーション備考
お名前String○1〜50文字、前後空白除去保護者名入力も可
ふりがなString○1〜80文字、ひらがな推奨任意でカタカナ許容
メールアドレスString○メール形式、最大255文字返信先として利用
電話番号String○数字とハイフンのみ、10〜13桁緊急連絡用
希望コースString○選択式（子ども/大人/検定/体験）プルダウン
希望日時（第1希望）DateTime○現在日時より未来、営業時間内営業日判定あり
希望日時（第2希望）DateTime—第1希望と重複不可任意入力
経験有無String○選択式（初心者/経験あり）指導準備に使用
お問い合わせ内容String—最大1000文字自由記述
個人情報同意Boolean○チェック必須未チェックは送信不可

5 非機能要件

セキュリティ要件 必須
認証・認可・データ保護に関する要件

・全ページ HTTPS を必須とし、フォーム送信は暗号化通信のみ許可する。

・フォーム入力値はサーバー側で再バリデーションし、SQLインジェクション/XSS対策を実施する。

・個人情報（氏名、連絡先）は最小権限で管理し、閲覧権限を運営担当に限定する。

・スパム対策として reCAPTCHA または同等のBot対策を導入する。

・管理通知メールに機微情報を過剰に含めない（必要最小限のみ送付）。

・プライバシーポリシーへの同意取得と同意履歴の保持を行う。

対応環境 必須
対応ブラウザ・デバイス

・対応ブラウザ：Chrome / Safari / Edge / Firefox の各最新版とその1つ前のメジャーバージョン

・対応デバイス：PC（1280px以上推奨）、タブレット、スマートフォン（iOS/Android）

・レスポンシブ対応：360px〜1440px の主要幅でレイアウト破綻がないこと

・性能目標：主要ページのLCP 2.5秒以内（4G相当）、CLS 0.1以下

・可用性目標：月間稼働率 99.5%以上（計画停止を除く）

・アクセシビリティ：主要導線でキーボード操作可能、画像には代替テキストを付与
